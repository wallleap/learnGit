Git 学习

1、初始化环境

 下载 Git、注册 GitHub 等账号

      # 查看 git 版本
      git --version

 配置本地 Git 信息

  • git config --global user.name "Your Name"
  • git config --global user.email "email@example.com"
  •         # 查看配置信息（按 q 退出）
        git config --list

添加了 --global 是针对全局生效的，如果某个仓库需要单独的，只需在这个仓库中使用没有 --global 的命令配置一遍即可

 生成 keygen

ssh-keygen -t rsa -C "email@example.com"

或使用 ed25519 算法

ssh-keygen -t ed25519 -C "SSH Key"

将生成的 ~/.ssh/id_rsa.pub 中的公钥放到需要提交的服务器上（~/.ssh/authorized_keys 文件中）或者在线平台的设置 SSH keys 新建一个，拷贝公钥内容上去

 测试连接（例如 GitHub）

ssh -T git@github.com

输出如下内容就代表成功

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

2、了解 Git 基本命令

工作区（working tree），暂存区（index /stage），本地仓库（repository），远程仓库（remote）

 本地仓库

初始化本地仓库

git init

创建 .gitignore 文件

      touch .gitignore
      # 修改这个文件，将需要忽略的文件名/目录写入
      vi .gitignore

查看仓库中文件状态

      git status

 暂存区

添加当前目录下文件到暂存区

      git add .
      # 添加部分文件
      git add 文件名|文件夹
      # 删除部分文件
      git rm --cached 文件名|文件夹
      # 恢复指定文件 或用 restore
      git checkout 文件名|文件夹

将暂存区中的文件进行提交

      git commit -m "本次提交的说明"
      # 修正最近一次提交（会保留原提交内容，仅补充文件/修改提交信息）
      git commit --amend

 暂存修改

      git stash
      # 恢复并删除 stash
      git stash pop

 远程仓库

添加远程仓库

      git remote add origin(名字) 仓库链接(ssh或https)

查看远程仓库

      git remote -v
      # 删除某个远程连接
      git remote remove origin(名字)

推送本地代码到远程仓库

      git push origin main:main
      # origin 是远程仓库别名、前一个 main 是本地分支名、后一个 main 是远程分支名
      # 第一次可以多加一个参数 -u，将设置默认推送目标
      git push -u origin master
      # 还有其他参数，-f 可以强制推送，--all 推送所有分支，--tag 推送所有标签

拉取远程代码到工作区

      git pull origin main:main
      # origin 是远程仓库别名、前一个 main 是远程分支名、后一个 main 是本地分支名
      # --rebase 参数使用变基，-r 禁用变基强制使用合并
      # fetch 下载最新更改到本地仓库，但不自动合并到当前分支
      git fetch

克隆远程代码到本地仓库

      git clone 仓库链接 目录名
      # -b main 指定克隆分支，--depth 1 指定克隆深度

 查看文件修改内容

      git diff 文件名

 查看历史记录

      git log
      git log --pretty=oneline

 查看引用日志

      git reflog

 版本回退

      # 回退到上一个版本(Git 中 HEAD 代表当前版本)
      git reset --hard HEAD^
      # 回退到上上个版本
      git reset --hard HEAD^^
      # 回退到上100个版本
      git reset --hard HEAD~100
      # 根据版本号回退
      git reset --hard 版本号前几位或写全
      # 撤消指定版本
      git revert 版本号前几位或写全

 标签

      # 列出所有 tag
      git tag
      # 在当前版本新建 tag
      git tag 标签名
      # 在某个版本新建 tag
      git tag 标签名 版本号
      # 删除 tag
      git tag -d 标签名

 分支管理

        # 创建分支
        git branch 分支名
        # 切换分支
        git checkout 分支名
        git switch 分支名
        # 可以合写为一条
        git checkout -b 分支名
        git switch -c 分支名
        # 查看分支，-r 参数查看远程分支
        git branch
        # 合并分支到当前分支
        git merge 分支名
        # 合并某个版本到当前分支
        git cherry-pick 版本号
        # 删除分支
        git branch -d 分支名

3. 冲突解决

避免使用 merge，而默认的 git pull 为 git fetch + git merge

 手动使用 rebase

        # 先暂存代码
        git stash
        # 分开 fetch 并 rebase 或使用 git pull --rebase
        git fetch
        git rebase
        # 查看封存记录
        git stash list
        # 恢复最新变更，或使用 pop
        git stash apply
        # 恢复列表中指定变更
        git stash apply记录

4. 大小写问题

在大小写敏感的语言中，可能会出错，不建议修改默认配置

 直接先修改成其他的单词再修改成对应大小写

例如 base 先修改成 temp 再修改成 Base

之后再提交

5. 常用分支名

 核心主分支（仓库的基础分支，长期存在）

  • 生产环境代码 main / master
  • 开发完成的功能代码 develop / dev

 临时开发分支（完成特定任务后删除，短期存在）

  • 功能分支 feature/xxx 或 feat/xxx，开发新需求、新功能、新特性（基于 develop 分支创建，完成后合并回 develop）
  • 修复分支 bugfix/xxx 或 fix/xxx，基于 develop 分支创建，完成后合并回 develop
  • 线上紧急修复分支 hotfix/xxx 或 hf/xxx，基于 main 分支创建，完成后同时合并回 main 和 develop 分支
  • 发布分支 release/xxx 或 rel/xxx，基于 develop 分支创建，完成后合并回 main 和 develop

 其他特殊场景分支

  • 测试环境 test / testing
  • 项目文档 docs
  • 代码重构 refactor

6. Conventional Commits（约定式提交） 规范

 feat - 新增功能（feature）

  • feat: 新增用户登录功能
  • feat: 实现商品列表分页查询
  • feat: 添加微信支付回调接口

 fix - 修复 Bug

  • fix: 修复登录验证码不显示的问题
  • fix: 解决订单金额计算错误 Bug
  • fix: 修复线上支付失败的紧急问题

 docs - 文档更新

  • docs: 更新 README 中的部署步骤
  • docs: 补充接口文档的参数说明
  • docs: 完善代码中的注释说明

 style - 代码格式调整（不影响功能）

  • style: 格式化代码缩进（符合 ESLint 规范）
  • style: 调整变量命名，统一驼峰式
  • style: 删除多余的空行和注释

 refactor - 代码重构（不影响功能）

  • refactor: 重构用户服务模块，提取公共工具类
  • refactor: 优化接口请求封装，简化代码逻辑
  • refactor: 拆分超大组件，提高可维护性

 test - 测试相关

  • test: 为用户登录功能添加单元测试
  • test: 完善订单模块的集成测试用例
  • test: 修复测试代码中的断言错误

 chore - 构建 / 工程配置 / 辅助工具变更（不涉及业务代码）

  • chore: 升级 axios 依赖到 1.6.0 版本
  • chore: 配置 Webpack 打包优化参数
  • chore: 添加 .gitignore 忽略日志文件
  • chore: 更新 ESLint 配置规则

 perf - 性能优化

  • perf: 优化商品列表查询速度，添加缓存
  • perf: 减少 DOM 操作，提升页面渲染性能

 revert - 回滚提交

  • revert: 回滚 "feat: 新增用户登录功能"
  • revert: 撤销 fix: 修复订单金额计算错误 Bug

 ci - 持续集成 / 持续部署配置变更

  • ci: 配置 GitHub Actions 自动打包部署
  • ci: 调整 GitLab CI 测试步骤的超时时间
